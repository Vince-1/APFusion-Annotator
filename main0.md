# 生成一个可运行html网页文件，要求实现以下功能：
页面分成数据列表、图像显示、统计参数三部分
## 数据列表
将数据按PatienId列表显示，可点击选中，默认选中第一个，有更改未保存的数据高亮
## 图像显示
  1. 根据选中的数据，可视化并排显示当前数据前面和背面；
  2. 同时将原始标注和预测标注框以不同颜色显示在对应图上，原始标注框PR使用蓝色虚线，预测标注框GT使用红色实线；
  3. 鼠标移到框的位置时显示对应的编号和置信度，如果已经有标注TP/FN/FP，则同时显示
  4. 下方显示对应报告
  5. 将前面图像显示在第三列，并把原始标注和预测标注框同时显示在上面，同时将FN和FP高亮，并且将聚合的标注合成一个大标注框
## 统计参数
  1. TP，值为数组，其元素为GT和PR的数组对，GT为预测标记框索引编号数组，PR内为对应的原始标记框编号数组，需去重，首先读取已保存的json文件，json格式如后所示，显示当前选中的数据对应的值，如果没有文件或者没有读取到数据，则显示为空
  2. FN同上
  3. FP同上
  4. current precision,计算当前选中数据的precision
  5. current recall，计算当前选中数据的recall
  6. total precision, 计算所有已统计数据的precision
  7. total recall，计算所有已统计数据的recall
  8. 保存按钮，将已更改的数据写入json文件

  json格式：
{
  "TP":100,
  "FN":10,
  "FP":5,
  "Precision":0.9504,
  "Recall":0.9091,        
  "Patient0001":{
    "TP"[
            {
            "GT":[0,1],"PR":[0,1]
            }
        ],
    "FP":{},
    "FN":{}
  }
}


  ## 交互
  1. 所有标注框和标记之间的连线，仅在鼠标悬浮在标注框和标记上时显示，此时标记高亮，若鼠标位置同时在多个标注框内，则都显示
    2. 将TP/FN/FP换成按钮，点击时进入对应的选择模式，按钮为单选状态（同一时刻仅一个模式生效）
    3. 进入某个模式后（TP/FN/FP），页面需展示三部分状态：
      3.1 已保存配对列表：显示该模式下当前病人的所有数组对
      3.2 当前草稿配对：显示临时选中的GT数组和PR数组
      3.3 画布高亮状态：该模式下已保存配对中的标记和标注框均高亮，并显示连线
    4. 左键点击交互规则（模式内一致）：
      4.1 左键点击标记或标注框时，执行“选中/取消选中”切换
      4.2 点击预测标注框时，索引进入当前草稿的GT数组；再次点击则从GT数组移除
      4.3 点击原始标注框时，索引进入当前草稿的PR数组；再次点击则从PR数组移除
      4.4 当前草稿配对实时显示为：GT:[...] 与 PR:[...]
    5. 草稿操作按钮：
      5.1 确认配对：将当前草稿写入当前模式数组（追加一条对象 {"GT":[],"PR":[]} ）
      5.2 清空草稿：仅清除当前草稿，不影响已保存配对
      5.3 删除配对：在“已保存配对列表”中删除选中的某一条配对
    6. 模式约束（用于避免歧义）：
      6.1 TP模式：确认配对前要求GT和PR均非空
      6.2 FN模式：确认配对前要求GT为空、PR非空
      6.3 FP模式：确认配对前要求GT非空、PR为空
      6.4 若不满足模式约束，“确认配对”按钮置灰并提示约束原因
    7. 去重与唯一性规则：
      7.1 每次确认配对前，GT和PR数组内索引自动去重并升序
      7.2 同一模式下，同一索引默认不允许重复出现在多条配对中
      7.3 若点击到已被其它配对占用的索引，提示“取消本次选择”或“从原配对迁移到当前草稿”
    8. 可视化联动规则：
      8.1 悬浮（hover）用于临时连线预览与高亮，不改动数据
      8.2 点击（left click）用于数据编辑，改动当前草稿
      8.3 在“已保存配对列表”点击某条配对时，画布仅高亮该配对对应的GT/PR及其连线，便于复核
    9. 保存规则：
      9.1 点击“保存”时将当前病人的TP/FN/FP完整写入json
      9.2 病人切换前若存在未保存修改，列表中该病人高亮并弹出确认提示（保存/放弃/取消切换）
